# 多进程训练副本使用说明

**作者**: Dionysus  
**日期**: 2025-01-16

## 概述

为了解决Dymola仿真时编译文件冲突的问题，我们提供了一个副本创建脚本，可以创建完全独立的工作目录，每个目录都有自己的编译环境和保存路径。

## 核心问题解决

### Dymola编译文件冲突
- **问题**: Dymola仿真会在根目录生成编译文件（如 `dsmodel.c`, `dymosim.exe`, `dslog.txt` 等）
- **后果**: 多个进程同时运行时会相互覆盖这些文件，导致仿真失败
- **解决**: 创建完全独立的工作目录，每个进程在自己的目录中运行

## 快速开始

### 1. 创建副本

```bash
# 创建2个进程副本（默认）
python copy_for_multiprocess.py

# 创建4个进程副本
python copy_for_multiprocess.py --processes 4

# 指定源目录
python copy_for_multiprocess.py --source "路径/到/源目录" --processes 3
```

### 2. 启动训练

创建完成后，会生成如下目录结构：
```
parent_directory/
├── efficient_rl/                    # 原始目录
├── efficient_rl_process_0/           # 进程0副本
├── efficient_rl_process_1/           # 进程1副本
├── efficient_rl_process_2/           # 进程2副本
└── efficient_rl_process_3/           # 进程3副本
```

#### 方法1：使用启动脚本（推荐）
每个副本目录都包含启动脚本：
- **Windows**: 双击 `start_training_p0.bat`
- **PowerShell**: 双击 `start_training_p0.ps1`

#### 方法2：手动启动
在不同的终端中执行：
```bash
# 终端1
cd efficient_rl_process_0
python main_efficient.py --mode train

# 终端2
cd efficient_rl_process_1
python main_efficient.py --mode train

# 终端3
cd efficient_rl_process_2
python main_efficient.py --mode train
```

### 3. 清理副本

```bash
# 删除所有创建的副本目录
python copy_for_multiprocess.py --cleanup
```

## 功能特性

### 智能文件复制
- **包含**: Python文件、配置文件、Modelica文件、文档、Web界面
- **排除**: 训练结果、缓存文件、Dymola编译文件、大型数据文件
- **自动**: 创建独立的保存目录，避免结果覆盖

### 配置自动调整
- 每个副本都有独立的 `save_dir` 配置
- 支持为不同进程设置不同的超参数
- 自动生成启动脚本

### 默认配置变化
脚本会为不同进程设置不同的配置：
- **进程0**: 学习率 0.0003, 隐藏层 128（默认）
- **进程1**: 学习率 0.0001, 隐藏层 128（保守）
- **进程2**: 学习率 0.0005, 隐藏层 256（激进）
- **进程3**: 学习率 0.0002, 隐藏层 64（轻量）

## 注意事项

### 资源消耗
- **CPU**: 每个进程占用1-2个CPU核心
- **内存**: 每个进程约需要2-4GB内存
- **磁盘**: 每个副本约占用100-200MB空间

### 建议配置
- **2核CPU**: 最多1个进程
- **4核CPU**: 最多2个进程
- **8核CPU**: 最多3-4个进程
- **16核CPU**: 最多6-8个进程

### 故障排除

#### 问题1：虚拟环境未激活
**现象**: 提示找不到Python包
**解决**: 确保在父目录有 `.venv` 虚拟环境，或手动激活

#### 问题2：Dymola许可证冲突
**现象**: 多个进程中只有一个能正常仿真
**解决**: 检查Dymola许可证是否支持并发使用

#### 问题3：内存不足
**现象**: 进程崩溃或系统卡顿
**解决**: 减少并行进程数量，或增加系统内存

#### 问题4：磁盘空间不足
**现象**: 训练中断或保存失败
**解决**: 清理旧的训练结果，或使用更大的磁盘

## 高级用法

### 自定义配置变化
修改 `copy_for_multiprocess.py` 中的 `config_variations` 列表：

```python
config_variations = [
    {'learning_rate': 0.0003, 'batch_size': 64},
    {'learning_rate': 0.0001, 'batch_size': 128},
    {'learning_rate': 0.0005, 'batch_size': 32},
]
```

### 指定复制文件
修改 `get_files_to_copy()` 方法中的 `include_extensions` 和 `include_dirs`。

### 批量结果分析
训练完成后，可以编写脚本批量分析所有进程的结果：

```python
import glob
import pandas as pd

# 收集所有进程的统计数据
all_stats = []
for stats_file in glob.glob("efficient_rl_process_*/efficient_training_results_p*/stats/*.csv"):
    df = pd.read_csv(stats_file)
    process_id = stats_file.split('_')[3].split('/')[0]  # 提取进程ID
    df['process_id'] = process_id
    all_stats.append(df)

# 合并并分析
combined_stats = pd.concat(all_stats, ignore_index=True)
# 进行对比分析...
```
